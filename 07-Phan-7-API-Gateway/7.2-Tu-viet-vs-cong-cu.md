# Phần 7.2 — Tự viết (Express) vs Kong / Traefik / Nginx / KrakenD

> Ta đang tự viết gateway bằng Express. Khi nào nên tiếp tục, khi nào nên đổi sang gateway "đóng hộp"?

---

## 7.2.1 — Hai trường phái

- **Tự viết (Express/Fastify)**: gateway là *code của bạn*. Toàn quyền — muốn logic gì cũng nhét được
  (BFF phức tạp, transform tuỳ biến). Đổi lại: bạn **tự lo** hiệu năng, bảo mật, vận hành.
- **Gateway đóng hộp** (Kong, Traefik, Nginx, KrakenD, APISIX…): phần mềm chuyên dụng, cấu hình bằng
  file/API. Nhanh, đã tối ưu, nhiều plugin (auth, rate-limit, cache) sẵn. Đổi lại: **ít linh hoạt**
  cho logic lạ, phải học cấu hình riêng.

## 7.2.2 — Bảng chọn nhanh

| Tiêu chí | Tự viết Express | Nginx | Traefik | Kong | KrakenD |
| --- | --- | --- | --- | --- | --- |
| Học/hiểu bản chất | ★★★ | ★ | ★★ | ★ | ★ |
| Hiệu năng thô | ★ | ★★★ | ★★ | ★★ | ★★★ |
| Auto service discovery | tự làm | ✕ | ★★★ (Docker/k8s labels) | ★★ | ★★ |
| Plugin sẵn (auth, rate-limit) | tự cài | ít | vừa | ★★★ | ★★ |
| Aggregation/BFF phức tạp | ★★★ | ✕ | ✕ | hạn chế | ★★ (khai báo) |
| TLS/Let's Encrypt tự động | tự làm | thủ công | ★★★ | ★★ | ★★ |

## 7.2.3 — Khuyến nghị thực dụng

- **Học / prototype / cần BFF logic riêng** → **tự viết** (như tutorial này). Bạn *thấy* mọi thứ.
- **Production, cần TLS tự động + discovery theo container** → **Traefik** (gắn nhãn Docker/k8s là
  xong, rất hợp docker-compose/k8s).
- **Cần hệ sinh thái plugin/quản trị API bài bản** (API keys, quota, marketplace) → **Kong**.
- **Chỉ cần reverse proxy nhanh, tĩnh** → **Nginx**.
- **Aggregation khai báo hiệu năng cao, không viết code** → **KrakenD**.

> Mẫu hình hay gặp ở production: **Traefik/Nginx làm edge (TLS, routing thô)** đứng trước, rồi mới tới
> **gateway tự viết/BFF** cho logic ứng dụng. Không nhất thiết "một chọi một".

## 7.2.4 — Cùng một routing, viết bằng công cụ khác

Xem `reference/` cạnh file này: **`nginx.conf`** và **`traefik.yml`** làm ĐÚNG việc gateway-lite của
ta (route `/api/auth`→auth-service, `/api/users`→user-service) — để bạn đối chiếu "code Express" với
"cấu hình khai báo". Chúng chỉ để tham khảo, **không** nối vào docker-compose.

Nhận xét khi đọc:

- Express: routing là **code** (`app.use("/api/auth", proxy(...))`) → linh hoạt, nhưng dài dòng.
- Nginx/Traefik: routing là **khai báo** → ngắn gọn, nhưng muốn "verify JWT rồi gộp 2 API" thì đuối.

## 7.2.5 — Vì sao tutorial vẫn tự viết tiếp

Vì các bài sau (7.3 verify JWT + truyền context, 7.5 BFF aggregation) cần **logic ứng dụng** mà gateway
đóng hộp làm gượng ép. Tự viết giúp bạn hiểu *chuyện gì thực sự xảy ra* — sau này chuyển sang
Kong/Traefik bạn cũng biết mình đang cấu hình cái gì.
