# Phần 7.4 — CORS tập trung, Rate Limiting & Transform ở Gateway

> Ba việc "ngang" nữa dồn về gateway: quản CORS một nơi, chặn abuse ở biên, và chỉnh sửa nhẹ
> request/response.

---

## 7.4.1 — CORS: một nơi duy nhất

Trước 7.4, **mỗi service** tự `app.use(cors(...))`. Nhưng chỉ **gateway** mới thực sự nhận request từ
trình duyệt — service phía sau chỉ nhận request server-to-server (từ gateway). Vậy:

- **Gateway**: giữ CORS (cho phép origin `http://localhost:5173`, `credentials: true`).
- **auth-service / user-service**: **bỏ** `cors` — bớt cấu hình, bớt chỗ sai. (OAuth redirect là điều
  hướng top-level, không dính CORS; gọi API đều qua gateway.)

> Nguyên tắc: cấu hình *ngang* chỉ nên ở **một tầng** — tầng tiếp xúc thế giới bên ngoài.

## 7.4.2 — Rate limiting: chặn abuse ngay ở biên

Đặt ở gateway để lọc **trước khi** chạm service (service không phí tài nguyên cho request rác).

| Giới hạn | Phạm vi | Vì sao |
| --- | --- | --- |
| **300 req / phút / IP** | toàn bộ `/api` | chặn abuse chung |
| **10 lần / 15 phút / IP** | `/api/auth/login`, `/register` | chống **brute-force** mật khẩu |

**Lưu bộ đếm ở Redis** (không phải bộ nhớ tiến trình): khi chạy **nhiều gateway** (scale ngang), mọi
instance chia sẻ **cùng** bộ đếm → giới hạn đúng tổng thể. Nếu để in-memory, mỗi instance đếm riêng →
kẻ tấn công lách bằng cách rải request qua nhiều instance.

> Nhớ Phần 2? Rate limit từng nằm trong monolith. Nay nó là việc của **biên**, không phải của service.
> Đây là ví dụ điển hình "mối quan tâm ngang thuộc về gateway".

## 7.4.3 — Transform request/response

Gateway là chỗ tự nhiên để "nắn" request/response:

- Ta thêm header `x-gateway: fs-gateway` cho response (đánh dấu đã qua gateway).
- `helmet` gỡ/khoá các header lộ thông tin (`x-powered-by`…).
- (Đã có từ 7.3) **inject** `x-user-*` vào request đi xuống service; **strip** header giả mạo ở đầu vào.

Thực tế transform còn dùng để: đổi version path (`/v1`→nội bộ), che field nhạy cảm, nén, đổi định dạng…

## 7.4.4 — Thứ tự middleware (quan trọng)

```
helmet → cors → requestId → log → x-gateway header
      → generalLimiter (/api) → authLimiter (/api/auth/login|register)
      → stripSpoofedHeaders → attachUser
      → requireAuth (route cần đăng nhập) → proxy
```

Rate limit đặt **sớm** (trước cả verify JWT) — chặn lũ request rác *trước khi* tốn công verify.

## 7.4.5 — Thử

```bash
# Spam login sai -> sau 10 lần trong 15' sẽ nhận 429 Too Many Requests.
for i in $(seq 1 12); do
  curl -s -o /dev/null -w "%{http_code}\n" -X POST http://localhost:4000/api/auth/login \
    -H 'Content-Type: application/json' -d '{"email":"x@y.z","password":"wrong"}'
done
# ...401 401 ... rồi 429 (rate limited). Header RateLimit-* cho biết còn bao nhiêu lượt.
```

> Tiếp theo **7.5 (cuối Phần 7)**: **BFF** — gateway tự gộp nhiều service thành một response gọn cho FE.
