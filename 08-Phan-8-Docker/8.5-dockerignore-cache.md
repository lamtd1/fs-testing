# Phần 8.5 — `.dockerignore` & tối ưu layer cache

> Build nhanh hơn + image nhỏ hơn. Các Dockerfile đã commit ưu tiên **đơn giản, chạy được** (đã
> verify); mục này chỉ ra cách **tối ưu** khi bạn muốn.

---

## 8.5.1 — `.dockerignore`: giữ context nhỏ + tránh "nhiễm" file host

`docker build` gửi **toàn bộ context** cho daemon. Không loại trừ thì `node_modules` (hàng trăm MB)
bị gửi đi → chậm, và tệ hơn: `COPY . .` kéo cả `node_modules`/`src/generated` của **host** vào image
→ sai nền (macOS/arm) khi image là Linux.

`code/.dockerignore` loại:

```
node_modules, **/node_modules      # image tự cài lại (đúng nền)
**/src/generated                   # Prisma client tự generate trong image (đúng engine Linux)
.env, **/.env                      # secret KHÔNG bao giờ vào image
**/dist, **/*.log, .DS_Store
```

## 8.5.2 — Layer cache: thứ tự COPY quyết định tốc độ rebuild

Docker cache theo **từng lệnh**; một lệnh đổi input → lệnh đó và **mọi lệnh sau** phải chạy lại.

Dockerfile đã commit (đơn giản):

```dockerfile
COPY . .                       # đổi 1 dòng code -> layer này đổi...
RUN pnpm install               # ...nên install CHẠY LẠI (chậm) dù deps không đổi
```

Tối ưu — **copy manifest trước, cài, rồi mới copy source**:

```dockerfile
# 1) chỉ manifest -> cache install bền vững khi sửa code
COPY pnpm-workspace.yaml pnpm-lock.yaml package.json tsconfig.base.json ./
COPY packages/shared/package.json      packages/shared/
COPY apps/api/package.json             apps/api/
COPY apps/auth-service/package.json    apps/auth-service/
COPY apps/user-service/package.json    apps/user-service/
COPY apps/notification-service/package.json apps/notification-service/
COPY apps/web/package.json             apps/web/
RUN pnpm install --frozen-lockfile --ignore-scripts   # (*)
# 2) rồi mới copy toàn bộ source
COPY . .
RUN pnpm --filter @app/user-service exec prisma generate
```

Giờ sửa code → `pnpm install` **dùng lại cache** (deps không đổi) → rebuild nhanh hơn nhiều.

> **(*) `--ignore-scripts` quan trọng**: auth/user có `postinstall: prisma generate`. Nếu cài khi CHƯA
> copy `schema.prisma` (bước manifest) thì postinstall generate sẽ **lỗi**. Nên bỏ qua scripts lúc
> cài, rồi generate **sau** khi đã copy source (dòng cuối).

## 8.5.3 — Image nhỏ hơn nữa: bundler

Hiện runner copy cả `node_modules` + source + `tsx` (devDep) → ảnh khá to. Muốn nhỏ thật:

- **Bundle** service thành 1 file JS bằng `esbuild`/`tsup` (gộp cả `@app/shared`), rồi image runner chỉ
  cần `node dist/index.js` — **không** pnpm, **không** node_modules, **không** tsx.
- Hoặc `pnpm deploy --prod --filter <svc> /out` để tạo thư mục self-contained chỉ gồm prod deps.

Với bundler, runner có thể mỏng như:

```dockerfile
FROM node:20-slim AS runner
COPY --from=builder /repo/apps/user-service/dist ./dist
COPY --from=builder /repo/apps/user-service/src/generated ./src/generated  # prisma engine
CMD ["node", "dist/index.js"]
```

> Tutorial dừng ở `tsx` cho đơn giản/minh bạch; bundler là bước tối ưu production đáng làm sau.

## 8.5.4 — Vài mẹo khác

- **Gộp `RUN apt-get`** + `rm -rf /var/lib/apt/lists/*` trong CÙNG một lệnh (đã làm) → không để cache
  apt phình layer.
- **Pin phiên bản base image** (`node:20-slim`, `nginx:1.27-alpine`) → build lặp lại ổn định.
- **Multi-stage**: công cụ build (pnpm, devDeps) nằm ở `builder`, không lọt vào ảnh chạy khi dùng bundler.
- **BuildKit cache mount** (nâng cao): `RUN --mount=type=cache,target=/root/.local/share/pnpm/store pnpm install`
  → cache store pnpm giữa các build, cực nhanh.
