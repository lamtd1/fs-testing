# Phần 8.2 — Dockerfile cho React (build tĩnh + Nginx)

> Frontend khác backend: sau khi build, nó chỉ là **file tĩnh** (HTML/CSS/JS). Không cần Node ở
> runtime — chỉ cần một web server phục vụ file + proxy `/api`. ✅ **build-verified**.

---

## 8.2.1 — Hai giai đoạn: build bằng Node, phục vụ bằng Nginx

```dockerfile
FROM node:20-slim AS builder     # vite build -> apps/web/dist
...
FROM nginx:1.27-alpine AS runner # chỉ Nginx + dist
COPY --from=builder /repo/apps/web/dist /usr/share/nginx/html
```

Ảnh cuối **không chứa Node/pnpm/source** — chỉ Nginx + thư mục `dist`. Vì thế nó **rất nhỏ** (vài chục
MB) và khởi động tức thì. Đây là điểm khác cốt lõi với backend (backend cần Node để chạy).

## 8.2.2 — Nginx làm hai việc

Xem `apps/web/nginx.conf`:

1. **SPA fallback**: `try_files $uri $uri/ /index.html`. React Router route phía client — khi user F5
   ở `/dashboard`, Nginx không có file `/dashboard` nên **trả `index.html`**, để React Router xử lý.
   Thiếu dòng này = F5 ở route con → 404.
2. **Proxy `/api`**: frontend gọi đường dẫn **tương đối** `/api/...` (xem `lib/api.ts` — `baseURL: ""`),
   Nginx chuyển tiếp sang `http://gateway:4000`. Nhờ vậy **cùng origin** → không dính CORS, và FE
   không cần biết địa chỉ gateway.

> `gateway` ở đây là **tên service trong docker-compose** (8.3) — Docker DNS tự phân giải thành IP
> container. Không hardcode IP.

## 8.2.3 — Vì sao build tương đối `/api` lại quan trọng

Frontend build ra file tĩnh **một lần**, chạy ở mọi môi trường. Nếu hardcode `http://localhost:4000`
vào bundle thì lên production (domain thật) là hỏng. Dùng `/api` tương đối → Nginx/hạ tầng quyết định
"api thật ở đâu" lúc chạy → **một bản build, chạy mọi nơi**.

## 8.2.4 — Thử

```bash
cd code
docker build -f apps/web/Dockerfile -t web .
# (cần gateway chạy để /api hoạt động — xem 8.3)
docker run --rm -p 8080:80 web    # mở http://localhost:8080
```

> Tiếp theo **8.3-8.4**: `docker-compose` dựng **toàn hệ thống** (web + gateway + 3 service + Postgres
> + Redis + Keycloak + Jaeger) chỉ bằng một lệnh, kèm healthcheck/mạng/biến môi trường.
