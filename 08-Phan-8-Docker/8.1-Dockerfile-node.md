# Phần 8.1 — Dockerfile chuẩn cho Node service

> Đóng gói từng service thành image chạy được ở bất kỳ đâu. Có ✅ **build-verified**: image
> `user-service` và `gateway` build thành công và container khởi động (HTTP + gRPC lắng nghe).

---

## 8.1.1 — Build context là `code/`, không phải từng thư mục service

Mỗi service phụ thuộc `packages/shared` và cần `pnpm-lock.yaml` chung. Nên **build context là `code/`**
(gốc workspace), `dockerfile` trỏ tới `apps/<svc>/Dockerfile`. Nhờ vậy image thấy được cả shared + lock.

## 8.1.2 — Multi-stage: builder → runner

```dockerfile
FROM node:20-slim AS builder      # cài deps + generate Prisma
...
FROM node:20-slim AS runner       # chỉ mang những gì cần để chạy
COPY --from=builder /repo /repo
```

- **builder**: `pnpm install --frozen-lockfile` + `prisma generate`.
- **runner**: nhận kết quả từ builder rồi chạy. Multi-stage giúp tách "công cụ build" khỏi "ảnh chạy"
  (ở đây ta copy cả cây để giữ symlink của pnpm — xem 8.5 về cách làm image NHỎ hơn bằng bundler).

## 8.1.3 — Ba quyết định quan trọng

1. **`node:20-slim` (Debian) thay vì alpine** — Prisma engine cần `libssl`; slim + `openssl` chạy ổn,
   tránh đau đầu musl của alpine.
2. **Generate Prisma client TRONG image** (không copy từ host) — engine là **nhị phân theo nền**;
   host là macOS/arm, image là Linux → phải sinh lại đúng engine Linux. (Vì thế `.dockerignore` loại
   `src/generated`.)
3. **Chạy production bằng `tsx`, không `tsc`** — `@app/shared` được import dạng **TS-source** (exports
   trỏ `./src/*.ts`); biên dịch `tsc` từng package rồi chạy `node dist` sẽ không phân giải được shared.
   `tsx` transpile lúc chạy → né sạch. Đổi lại: khởi động chậm hơn chút + ảnh mang cả devDeps (tsx).
   8.5 nói cách tối ưu (bundler → 1 file JS → ảnh nhỏ, khởi động nhanh).

## 8.1.4 — `corepack prepare` để không tải pnpm lúc chạy

`RUN corepack enable && corepack prepare pnpm@9.12.0 --activate` — **bake** đúng phiên bản pnpm vào
image. Nếu chỉ `corepack enable`, lần chạy đầu container sẽ **tải pnpm từ mạng** (chậm, và hỏng nếu
offline).

## 8.1.5 — Bốn service, hai kiểu

| Service | Prisma? | EXPOSE | Ghi chú |
| --- | --- | --- | --- |
| auth-service | ✅ | 4001 | generate + openssl |
| user-service | ✅ | 4002 | generate + openssl |
| api (gateway) | ✕ | 4000 | không cần openssl |
| notification-service | ✕ | — | worker, không mở port |

## 8.1.6 — Thử build tay

```bash
cd code
docker build -f apps/user-service/Dockerfile -t user-service .
docker run --rm -e DATABASE_URL=... -e GATEWAY_SECRET=... -e OTEL_SDK_DISABLED=true user-service
#  -> "🚀 user-service (HTTP) tại http://localhost:4002" + "🔌 gRPC :4092"
```

> Ở 8.3 ta không build tay từng cái nữa — `docker compose` lo hết. Tiếp theo **8.2**: đóng gói
> frontend React (build tĩnh + Nginx serve).
